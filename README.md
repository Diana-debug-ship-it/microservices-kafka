# 🛒 Микросервисная система заказа и склада через Apache Kafka

Это микросервисная экосистема на **Spring Boot**, **Apache Kafka** и **PostgreSQL**, реализующая асинхронный событийно-ориентированный процесс создания заказов и автоматического списания товаров на складе (паттерн Сага / Хореография).

## 🛠️ Технологический стек
* **Бэкенд:** Java 17+, Spring Boot 3.x, Spring Data JPA, Lombok
* **Брокер сообщений:** Apache Kafka (Spring for Apache Kafka)
* **База данных:** PostgreSQL (изолированные БД для каждого сервиса)

---

## 🏗️ Архитектура системы

Проект построен по принципу слабой связанности. Сервисы общаются исключительно через топики Kafka:

1. **Сервис заказов (`order-service`)** — принимает запросы от клиентов, сохраняет заказ в своей БД со статусом `CREATED`, отправляет событие в Kafka и ожидает ответа для обновления финального статуса.
2. **Сервис склада (`warehouse-service`)** — слушает топик новых заказов, проверяет остатки и цены в своей БД, списывает товар и отправляет результат (`APPROVED` / `REJECTED`) обратно через Kafka.

### 🔄 Схема движения данных
```text
[Клиент/Postman] ➡️ (POST) ➡️ [Сервис Заказов] (Status: CREATED)
                                    │
                             (Топик "orders")
                                    ▼
                             [Apache Kafka]
                                    │
                        (Топик "order-statuses")
                                    ▼
[Клиент/Postman] ⬅️ (GET) ⬅️ [Сервис Заказов] (Status: APPROVED/REJECTED)
```

## 🧪 Примеры запросов (Postman)

### Создание заказа
* **Метод:** `POST`
* **URL:** `http://localhost:8080/orders`
* **Body (JSON):**
```json
{
  "productId": 1,
  "quantity": 2
}
```

### Проверка статуса и итоговой цены
* **Метод:** `GET`
* **URL:** `http://localhost:8080/orders/{id}`
